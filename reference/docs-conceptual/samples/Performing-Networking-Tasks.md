## <a name="pinging-computers"></a><span data-ttu-id="edf22-118">Ping 计算机</span><span class="sxs-lookup"><span data-stu-id="edf22-118">Pinging Computers</span></span>

<span data-ttu-id="edf22-121">摘要信息是更为有用的形式，它显示下面的命令生成的 Address、ResponseTime 以及 StatusCode 属性。</span><span class="sxs-lookup"><span data-stu-id="edf22-121">A more useful form for summary information a display of the Address, ResponseTime, and StatusCode properties, as generated by the following command.</span></span> <span data-ttu-id="edf22-122">Format-Table 的 Autosize 参数调整表列的大小，以使其正确显示在 Windows PowerShell 中。</span><span class="sxs-lookup"><span data-stu-id="edf22-122">The Autosize parameter of Format-Table resizes the table columns so that they display properly in Windows PowerShell.</span></span>

<span data-ttu-id="edf22-123">如果 StatusCode 为 0，指明 ping 操作成功。</span><span class="sxs-lookup"><span data-stu-id="edf22-123">A StatusCode of 0 indicates a successful ping.</span></span>

<span data-ttu-id="edf22-126">可以使用相同的命令格式对一个子网（例如使用网络号码 (192.168.1.0) 和标准 C 类子网掩码 (255.255.255.0) 的专用网）上的所有计算机执行 Ping 操作。仅在 192.168.1.1 到 192.168.1.254 范围内的地址为合法本地地址（0 始终为网络号码保留，255 是子网广播地址）。</span><span class="sxs-lookup"><span data-stu-id="edf22-126">You can use the same command format to ping all of the computers on a subnet, such as a private network that uses network number 192.168.1.0 and a standard Class C subnet mask (255.255.255.0)., Only addresses in the range of 192.168.1.1 through 192.168.1.254 are legitimate local addresses (0 is always reserved for the network number and 255 is a subnet broadcast address).</span></span>

<span data-ttu-id="edf22-129">请注意，这一用于生成一系列地址的方法也可用于其他地方。</span><span class="sxs-lookup"><span data-stu-id="edf22-129">Note that this technique for generating a range of addresses can be used elsewhere as well.</span></span> <span data-ttu-id="edf22-130">你可以使用以下方式生成完整的地址集：</span><span class="sxs-lookup"><span data-stu-id="edf22-130">You can generate a complete set of addresses in this way:</span></span>
